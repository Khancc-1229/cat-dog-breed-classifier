# 🐱🐶 猫狗品种识别

基于**两阶段级联架构**的猫狗品种细粒度分类系统，含拒识机制与持续迭代优化。

![界面截图](outputs/screenshot.png)

## 🎯 项目概述

| 维度 | 内容 |
|------|------|
| 任务 | 细粒度图像分类 —— 15 种常见猫狗品种识别 |
| 数据 | 自建混合数据集（Roboflow + Stanford Dogs + 百度爬虫人工筛选），5000+ 张 |
| 方法 | ImageNet 预训练权重 + 迁移学习微调 |
| 架构 | 两阶段级联：yolov8n.pt 检测定位 → YOLOv8-cls vs ResNet50 双模型品种分类 |
| 对比 | 轻量现代架构 YOLOv8-cls (2.7M 参数) vs 经典深网 ResNet50 (25M 参数) |
| 额外 | 置信度拒识机制：不在识别范围内的品种显示"可能不在识别范围内" |
| 部署 | Gradio Web 即时推理 |

## 🏗️ 两阶段级联架构

```
输入图片（可能含多只猫狗）
    │
    ▼
┌─────────────────────┐
│  Stage 1: 检测       │  yolov8n.pt (COCO 预训练)
│  框出图中所有猫/狗    │  开箱即用，无需标注任何框
│  过滤置信度 < 30% 的  │
└──────────┬──────────┘
           │ 对每个检测框逐张裁剪
           ▼
┌─────────────────────┐
│  Stage 2: 品种分类   │  YOLOv8-cls vs ResNet50
│  双模型同时预测       │  各自返回品种 + 置信度
│  置信度 < 阈值 → 拒识 │
└──────────┬──────────┘
           │
           ▼
     画框标注 + 双模型对比 + 一致性判断
```

### 为什么不是端到端检测？

- 端到端需要为每张训练图标注边界框坐标（labelme 画 5000 个框 = 3-5 天全职）
- 级联方案 Stage 1 用 COCO 预训练检测器，完全不用自己标框
- Stage 2 只需要文件夹级标签（品种名 = 文件夹名），成本为零
- 两个模块解耦，各自可独立替换或升级

- **难易搭配**：哈士奇、柯基等特征鲜明 + 阿拉斯加/哈士奇、美短/狸花猫等经典易混淆对子

## 🔄 迭代优化记录

### v1 → v2：发现美短召回率低 → 补数据 → 验证

```
v1 (初始训练)
  │
  ├─ 混淆矩阵分析 → 美短被大量误判为狸花猫
  │   YOLOv8 美短召回 73%，ResNet50 仅 59%
  │
  ├─ 根因定位 → 美短 220 张 vs 狸花猫 476 张，样本不平衡
  │
  ├─ 修复方案 → 百度爬虫多关键词搜索 + 人工逐张筛选
  │   美短补至 500+ 张，与狸花猫基本 1:1
  │
  └─ v2 验证
```

| 指标 | v1 YOLO | v1 ResNet | v2 YOLO | v2 ResNet | 变化 |
|------|:-------:|:---------:|:-------:|:---------:|:----:|
| Top-1 | 90.52% | 90.73% | 90.39% | **93.14%** | ResNet +2.4 |
| 美短召回 | 73% | 59% | **89%** | **86%** | +16 / +27 |
| 美短 F1 | 0.74 | 0.67 | 0.84 | 0.87 | |

### 关键洞察

- **YOLOv8 Top-1 微降 0.13% 不是退步**：补数据后模型从"偏科"变"均衡"，狸花猫、柴犬等之前虚高的品种回归真实水平，整体更健康
- **ResNet50 在 v2 反超 YOLOv8**：数据量充足后，深网的表达能力优势开始显现
- **补数据的收益是非线性的**：美短从 220→500 的提升远超预期，尤其是 ResNet50（+27pp）
- **数据质量 > 数量**：人工筛选爬虫图片是 v2 成功的关键，直接混入的杂质会抵消补数据的收益

## 🛡️ 拒识机制

### 为什么需要拒识

分类模型没有"不知道"这个选项——遇到不在 15 种里的猫狗，它也会强制选一个最近似的品种。解决方案：

1. 收集 7 种不在册品种（斑点狗、斯芬克斯猫、波斯猫等）作为负样本
2. 分析在册/不在册的 Top-1 置信度分布
3. 找到能最好区分两者的阈值
4. 低于阈值 → 拒绝输出，显示"可能不在识别范围内"

### 阈值分析结果

| 模型 | 阈值 | 在册误拒 | 不在册误收 |
|------|:----:|:--------:|:----------:|
| YOLOv8-cls | 0.92 | 15.5% | 37.1% |
| ResNet50 | 0.94 | 12.7% | 47.2% |

### 发现

- 不在册品种平均置信度高达 80%，与在册品种分布高度重叠
- 阈值设置在重叠区中间（误差和最小原则）
- 这不是完美的拒识，但比没有拒识好：至少 63% 的未知品种能被正确拦住
- 根本解决需要训练专门的"其他"类别或开放集识别方法

## 📦 获取数据

数据集已上传到 [GitHub Releases](https://github.com/Khancc-1229/cat-dog-breed-classifier/releases/tag/data-v1.0)，下载 `data.tar.gz.partaa` 和 `data.tar.gz.partab`（两个分片共 3.1GB）后：

```bash
# 合并分片
cat data.tar.gz.partaa data.tar.gz.partab > data.tar.gz

# 解压到项目根目录
tar -xzf data.tar.gz
```

即可还原 `data/` 文件夹。

同时下载 `yolov8_best.pt` 和 `resnet50_best.pth`，放到 `models/` 下：

```bash
# 放置模型权重
mkdir -p models/yolov8_cls_breeds/weights
mv yolov8_best.pt models/yolov8_cls_breeds/weights/best.pt
mkdir -p models/resnet50_breeds
mv resnet50_best.pth models/resnet50_breeds/best.pth
```

## 🚧 后续改进方向

### 训练前预裁剪

当前训练数据是整张原始图片（含背景），训练时模型学到的特征混合了动物+环境。更好的做法：

```
训练图 → yolov8n.pt 框出动物 → 只保留框内像素 → 送入分类模型训练
```

这样模型只能学到动物本身的特征，不会被背景干扰。这是典型的**检测→分类级联**思路在训练阶段的延伸——当前项目只在推理阶段做了级联，训练阶段可以进一步强化。

### 端到端检测对比

当前是分类模型（文件夹名=标签，零标注成本）。端到端检测需要在每张图上标注边界框+品种（5000 张图 x 手动画框 = 巨大工作量）。可以拿少量标注数据做检测版本的对比实验，分析精度/成本 tradeoff。

### 更严肃的拒识方案

当前阈值法是一种权衡——在册/不在册分布重叠严重。更根本的方案是在训练集中加入「其他品种」类别，或用 Open-Set Recognition 方法让模型学会表达"我不认识"。

## 🐛 遇到的问题和解决方案

| 问题 | 原因 | 解决 |
|------|------|------|
| Bing 正则匹配不到图片URL | Bing 用 `&quot;` 编码引号，标准正则 `"murl"` 匹配失败 | `text.replace("&quot;", '"')` 后再匹配 |
| 清洗时 332/500 张被判定太小 | MIN_SIZE=224 太严格，275×183 等长方形图一维超了就丢 | 降到 100，训练时 resize 会处理 |
| YOLOv8 Top-1 99.4% → 实际是二分类 | 数据集结构 `train/狗/哈士奇/` 让 YOLOv8 只看到"狗""猫"两层 | 扁平化结构 `train/哈士奇/`，重训后正常 |
| YOLOv8 评估报 `index 12 out of bounds for size 3` | `model.val()` 的内部类名映射与新数据集结构冲突 | 改为逐张推理，手动对齐类名 |
| ResNet 训练只到 epoch 6 就停了 | PATIENCE=5，新分类头冷启动需要更多时间 | PATIENCE 改为 12 |
| ResNet Epoch 1 准确率只有 64% | 分类头随机初始化（冷启动），浅层虽预训练但顶层从零学 | 正常现象，5-10 轮后会追上（最终 93%） |
| 美短、狸花猫严重混淆 | 美短 220 张 vs 狸花猫 476 张，模型偏向多数类 | 补美短至 500+，召回从 59%→86% |
| 金渐层被强制归类为布偶猫 | 模型没有"其他"选项 | 置信度阈值拒识机制 |
| Gradio 中文显示为方块 | matplotlib 默认无中文字体 | 指定微软雅黑字体路径 |
